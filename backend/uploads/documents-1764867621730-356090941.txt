# 2) Frontend design & rules (what users see before/after login)

**Global rules you should give Copilot / developer team:**

* **No emojis** in official UI copy. Use a consistent icon set (Material Icons, Heroicons, or FontAwesome Pro). Example: use `<svg>` official icons from the chosen library.
* **Typography**: Use variable fonts (Inter/InterVariable) for headings/body. Maintain consistent scale (H1, H2, H3).
* **Color system**: Define core semantic variables: `--brand`, `--accent`, `--bg`, `--muted`, `--text` and `--danger`. Do not hardcode colors inline.
* **Accessible contrast**: Ensure text 4.5:1 for body and 3:1 for large headings in default theme.
* **Microcopy**: Use short, actionable CTA text. Avoid casual slang.
* **Icons instead of emojis**: Always pick icons that communicate meaning (download, settings, login, lock, lock-open). Use tooltips.

**Pages / views and what they show**

1. **Public (logged-out)**

   * Homepage hero, Agents gallery (cards with short description), Pricing / Plans teaser, Features, Footer.
   * Signup / Login CTA prominent.
   * **No access** to agent run endpoints (calls requiring backend auth must return 401).
   * Allow: read-only agent detail pages with demo mode (sandbox prompt) and sample outputs — but actual run -> show modal asking to log in.
   * Allow download of public docs, FAQ, and contact form.

2. **Authenticated (logged-in)**

   * Dashboard with recent runs, saved projects, billing, API keys, and full access to run agents.
   * Per-agent detail page: full prompt editor, config options, run history, download results, schedule runs.
   * Role-based views (user vs admin): admin can manage agent configs, view logs, throttle, toggle public/private.

3. **Safety gating**

   * For agents that can take actions (send emails, trade, download), require **explicit consent and authorization** (OAuth screens or API key linkage).
   * Show clear permission scopes when connecting external accounts.

4. **Errors & limits**

   * Show clear, friendly errors: e.g., “Rate limit exceeded — try again in 30s” with a suggested action.
   * Provide “Sandbox mode” for demo when user is not logged in (no external calls, fake data) — label clearly.

# 3) Theme switching (Dark/Light) — plan + code

**Approach summary**

* Use CSS variables for colors and define two sets: `:root` (light) and `.theme-dark`.
* Use `prefers-color-scheme` to detect default and fallback to user preference stored in `localStorage`.
* Provide accessible color tokens and let Tailwind map to CSS variables or use Tailwind with CSS variables.

**Why this is crucial**

* Color consistency and contrast are easier when you reference tokens (brand/primary/secondary/text/muted). Theme toggle only swaps tokens.
* Use official brand colors for accents and keep background/foreground neutral.

**React + Tailwind friendly snippet (give to Copilot)**

```jsx
// ui/ThemeProvider.jsx
import React, { useEffect, useState } from 'react';

export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState(
    localStorage.getItem('site-theme') ||
    (window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light')
  );

  useEffect(() => {
    document.documentElement.classList.toggle('theme-dark', theme === 'dark');
    localStorage.setItem('site-theme', theme);
  }, [theme]);

  return children;
};

// Toggle button usage:
<button
  aria-label="Toggle theme"
  onClick={() => setTheme(prev => prev === 'dark' ? 'light' : 'dark')}
>
  <svg ...>/* use official icon */</svg>
</button>
```

**CSS variables (global.css)**

```css
:root{
  --bg: #ffffff;
  --card-bg: #ffffff;
  --text: #0f1724;
  --muted: #6b7280;
  --brand: #ff8a00; /* orange accent */
  --accent-2: #ffbf69;
  --surface-elev-1: #f7f7f8;
}

.theme-dark{
  --bg: #0b1220;
  --card-bg: #0f1724;
  --text: #e6eef6;
  --muted: #9aa6b2;
  --brand: #ff9f1c;
  --accent-2: #ffbf69;
  --surface-elev-1: #0b1220;
}

body{
  background: var(--bg);
  color: var(--text);
}
.card{ background: var(--card-bg); box-shadow: 0 6px 18px rgba(2,6,23,0.08); }
.btn-primary{ background: var(--brand); color: white; }
```

**Tailwind note**

* Use Tailwind CSS but reference CSS variables in `tailwind.config.js` or use `style={{ backgroundColor: 'var(--card-bg)' }}` in components.
* Or use Twin.macro / plugin that maps CSS variables to Tailwind tokens.

# 4) Icons & no emoji rule (brief directive for Copilot/devs)

* Do **not** insert emoji characters anywhere in UI copy, navigation, or buttons.
* Use a single icon library across the site: **Heroicons (outline/solid)** or **Material Icons**. Example: `<HeroIcon name="UserIcon" />`.
* Provide an icons component wrapper that maps semantic names to svg:

  ```jsx
  // ui/Icon.jsx
  import { UserIcon, LockClosedIcon } from '@heroicons/react/outline';
  const ICONS = { user: UserIcon, lock: LockClosedIcon, download: /* ... */ };
  export default function Icon({ name, className }) {
    const C = ICONS[name];
    return C ? <C className={className} aria-hidden="true"/> : null;
  }
  ```

### B) Agent lifecycle (run a task)

```
[User clicks RUN] -> POST /api/agents/:slug/run  (auth token)
     |
     v
[API validates input] -> creates Job record -> push Job -> Queue
     |
     v
[Worker picks Job] -> tool chain: LLM -> external API -> process -> save results
     |
     v
[Worker updates Job status] -> send progress via WS -> result stored in DB/S3
     |
     v
[Frontend receives WS event] -> shows result preview / download link
```
